## 메모리의 이해

메모리는 CPU와 저장장치 사이에서 데이터를 빠르게 읽고 쓰기 위해 존재하는 장치입니다. 메모리는 계층 구조로 구성되어 있으며, 각 계층은 속도와 용량에서 trade-off 관계를 가집니다.

- 레지스터(Register): CPU 내부에 존재하며, 가장 빠르지만 용량이 작음
- 캐시(Cache): CPU와 주기억장치 사이에 있는 고속 메모리
- 주기억장치(Main Memory, RAM): 실행 중인 프로그램과 데이터를 저장하는 장소
- 보조기억장치(Secondary Storage): 하드디스크, SSD 등 장기 저장용 장치

## 가상 메모리 관리

가상 메모리는 실제 물리 메모리보다 큰 메모리 공간을 제공하는 기술로, 각 프로세스는 독립된 메모리 공간을 갖습니다.

- 가상 주소: 프로세스가 사용하는 주소
- 물리 주소: 실제 RAM 상의 주소
- 페이지(Page): 가상 메모리는 일정한 크기(예: 4KB)로 나뉨
- 프레임(Frame): 물리 메모리도 페이지와 같은 크기로 나뉨

운영체제는 페이지 테이블을 이용하여 가상 주소를 물리 주소로 변환합니다.

### 가상 메모리의 장점
- 메모리 보호
- 프로세스 간 메모리 독립성 보장
- 물리 메모리보다 큰 공간 제공
- 단편화 문제 해결

## 페이지 교체

RAM에 빈 공간이 없을 때 새로운 페이지를 메모리에 적재하기 위해 기존 페이지를 제거하는 작업입니다.

### 페이지 교체 알고리즘

1. FIFO (First-In First-Out)  
   가장 먼저 들어온 페이지를 제거

2. LRU (Least Recently Used)  
   가장 오랫동안 사용되지 않은 페이지를 제거

3. Optimal  
   앞으로 가장 오랫동안 사용되지 않을 페이지 제거 (이론적 최적)

4. Clock (Second Chance)  
   FIFO에 LRU 개념을 결합한 방식

### 페이지 폴트 (Page Fault)
필요한 페이지가 메모리에 없는 경우 발생하며, 디스크에서 데이터를 불러오는 작업이 필요함. 잦은 페이지 폴트는 성능 저하의 원인이 되며, 이를 스래싱(Thrashing)이라고 함.
